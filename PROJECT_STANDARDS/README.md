# PROJECT_STANDARDS

该目录是 Ben_cloud 的 **新项目统一规范中心**。

唯一目标：

1. 未来新增子项目时，提供统一工程规范与落地清单。
2. 保证新项目与 Ben_cloud 现有体系在架构、迁移、测试与运维上保持一致。
3. 将实践沉淀为可复用标准，避免“每个新项目重新发明流程”。

非目标：

1. 不承担现有子项目的日常任务调度。
2. 不替代各子项目自己的 `AGENTS.md`。
3. 不提供按任务类型的角色画像体系。

目录说明：

- `PLAYBOOK.md`：新项目从立项到交付的搭建流程。
- `FASTAPI_ENGINEERING_STANDARD.md`：工程结构与分层硬约束。
- `FASTAPI_UNIFICATION_PROGRESS.md`：新项目落地检查清单。
- `IMPROVEMENT_LOOP.md`：标准文档自身的演进机制。
- `registry.yaml`：Ben_cloud 子项目注册表与新项目模板字段。
