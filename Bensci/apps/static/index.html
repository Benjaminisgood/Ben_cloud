1. 拉取并入库

系统会先尝试 AI 解析并翻译到各数据源语法，失败时自动回退为原词直接检索；返回结果可再按你选择的 query 复核模式逐篇判断。

建议写法：检索词尽量宽容（提高召回），把“严格标签判定”写到“仅复核提示”里让 LLM 只在复核阶段执行。

高级约束（默认折叠）

文章影响因子过滤只按当前文章记录里的 `impact_factor` 字段判断，不再按期刊映射表回填。

`embedding` 会先对 AI 提取出的 `must_terms` 做硬校验，过了再计算与 query 的余弦相似度，并把已入库 DOI 的向量缓存到 SQLite；`boolean` 按 AND / OR / NOT 严格过滤，前端会自动锁定阈值为 `1`；`llm` 会逐篇给出 0 到 1 的关联度分并按阈值保留，默认按当前 query 本身及其解析出的检索意图判断相关度；`none` 不做 query 复核，前端会自动锁定阈值为 `0`。

关键词约束是额外高级约束；即使 query 复核模式关闭，手动填写的必须/至少一个/排除关键词仍然生效。

若关闭“已入库 DOI 二次复核”，命中已入库文献时会直接入库更新标签；若关闭“已 drop DOI 二次复核”，命中历史 drop 文献时会直接跳过。
